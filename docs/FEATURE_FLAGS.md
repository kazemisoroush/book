# Feature Flags Catalog

This document catalogues all end-to-end features and their toggleability in the audiobook generator.

## Overview

Features fall into three categories:

1. **Explicitly Flagged** — Boolean constructor parameters that can be toggled at runtime
2. **Implicitly Controlled** — Features that require dependency injection or configuration changes
3. **Model-Gated** — Features controlled by ElevenLabs model selection (compile-time only)

---

## Explicitly Flagged Features

These features can be toggled on/off at runtime by passing constructor parameters to `TTSOrchestrator` or `TTSProjectGutenbergWorkflow`.

### Ambient Background Sound (US-011)

- **Flag**: `ambient_enabled: bool = True`
- **Location**: `TTSOrchestrator.__init__(ambient_enabled=True, ambient_client=None)`
- **Behavior**:
  - When `True` AND `ambient_client` is provided: generates ambient audio per scene via ElevenLabs Sound Effects API
  - When `False` OR `ambient_client is None`: skips ambient generation entirely
  - Ambient runs continuously at -28 dB under speech and sound effects
- **To Disable**:
  ```python
  orchestrator = TTSOrchestrator(..., ambient_enabled=False)
  # OR
  orchestrator = TTSOrchestrator(..., ambient_client=None)
  ```
- **Impact**: Scene ambient descriptions are generated by LLM but not synthesized if flag is off

### Sound Effects (US-023)

- **Flag**: `sound_effects_enabled: bool = True`
- **Location**: `TTSOrchestrator.__init__(sound_effects_enabled=True, sound_effect_client=None)`
- **Behavior**:
  - When `True` AND `sound_effect_client` is provided: inserts diegetic sound effects into inter-segment silence gaps
  - When `False` OR `sound_effect_client is None`: skips sound effects insertion entirely
  - sound effects is cached by description hash
- **To Disable**:
  ```python
  orchestrator = TTSOrchestrator(..., sound_effects_enabled=False)
  # OR
  orchestrator = TTSOrchestrator(..., sound_effect_client=None)
  ```
- **Impact**: Segment sound_effect_description fields are generated by LLM but not synthesized if flag is off

### Emotion-Aware TTS (US-009, US-010)

- **Flag**: `emotion_enabled: bool = True`
- **Location**: `TTSOrchestrator.__init__(emotion_enabled=True)` and `TTSProjectGutenbergWorkflow.run(emotion_enabled=True)`
- **Behavior**:
  - When `True`: LLM-detected emotion tags are used in TTS synthesis
  - When `False`: emotion tags are generated but not applied to voice synthesis
  - Scene context is still generated but emotion-based voice modifiers are ignored
- **To Disable**:
  ```python
  orchestrator = TTSOrchestrator(..., emotion_enabled=False)
  # OR via CLI:
  python scripts/run_workflow.py --url <url> --workflow tts --disable-emotion
  ```
- **Impact**: Disables emotion-based voice modulation; can improve consistency at cost of expression

### Character Voice Design (US-003, US-014)

- **Flag**: `voice_design_enabled: bool = True`
- **Location**: `TTSOrchestrator.__init__(voice_design_enabled=True)` and `TTSProjectGutenbergWorkflow.run(voice_design_enabled=True)`
- **Behavior**:
  - When `True`: VoiceAssigner calls Voice Design API for characters with descriptions
  - When `False`: Falls back to demographic (sex/age) matching; Voice Design API is not called
  - Voice design prompt is still generated but not used
- **To Disable**:
  ```python
  orchestrator = TTSOrchestrator(..., voice_design_enabled=False)
  # OR via CLI:
  python scripts/run_workflow.py --url <url> --workflow tts --disable-voice-design
  ```
- **Impact**: Saves API costs; all characters use demographic matching instead of custom voices

### Scene/Acoustic Context (US-020)

- **Flag**: `scene_context_enabled: bool = True`
- **Location**: `TTSOrchestrator.__init__(scene_context_enabled=True)` and `TTSProjectGutenbergWorkflow.run(scene_context_enabled=True)`
- **Behavior**:
  - When `True`: SegmentContextResolver looks up scene.voice_modifiers and applies them to voice settings
  - When `False`: Scene IDs are still assigned to segments but voice modifiers are not applied
  - Scene context is still generated by LLM
- **To Disable**:
  ```python
  orchestrator = TTSOrchestrator(..., scene_context_enabled=False)
  # OR via CLI:
  python scripts/run_workflow.py --url <url> --workflow tts --disable-scene-context
  ```
- **Impact**: Disables environment-aware voice adjustments; can simplify voice tuning

### Debug Mode

- **Flag**: `debug: bool = False`
- **Location**: `TTSOrchestrator.__init__(debug=False)` and `TTSProjectGutenbergWorkflow.run(debug=False)`
- **Behavior**:
  - When `True`: keeps individual segment MP3 files in `output_dir/segments/` alongside stitched chapter MP3
  - When `False`: deletes segment files after stitching
- **To Enable**:
  ```python
  orchestrator = TTSOrchestrator(..., debug=True)
  ```
- **Impact**: Useful for debugging TTS quality per segment; increases disk usage

## Implicitly Controlled Features

These features don't have explicit boolean flags. Instead, they're controlled by LLM behavior, dependency injection, or optional fields.

### LLM-Provided Voice Settings (US-019 Fix 3)

- **Status**: Enabled by default (LLM-driven)
- **Control**: No explicit flag; optional fields on Segment
- **Behavior**:
  - LLM returns `voice_stability`, `voice_style`, `voice_speed` per segment
  - Stored in `Segment.voice_stability/style/speed` (default `None`)
  - `TTSOrchestrator` passes to ElevenLabs if non-None
  - Scene modifiers adjust additively with clamping
- **To Disable**: Would require modifying AI parser prompt to NOT request voice settings

### Cross-Section Speaker Context

- **Status**: Enabled by default with 5-section window
- **Control**: `context_window` parameter to `AISectionParser`
- **Behavior**:
  - Parser receives up to 5 preceding sections for pronoun/speaker resolution
  - Noise-only sections (OTHER/ILLUSTRATION/COPYRIGHT) filtered before capping
- **To Disable**:
  ```python
  parser = AISectionParser(provider)
  segments, registry = parser.parse(section, registry, context_window=0)
  ```
- **Impact**: Less context for ambiguous speaker resolution

### Inter-Segment Silence (US-016)

- **Status**: Enabled by default
- **Control**: Constructor parameters `silence_same_speaker_ms`, `silence_speaker_change_ms`
- **Behavior**:
  - 150ms gap between same-speaker consecutive segments
  - 400ms gap at speaker boundaries
- **To Customize**:
  ```python
  orchestrator = TTSOrchestrator(
      ...,
      silence_same_speaker_ms=100,
      silence_speaker_change_ms=300
  )
  ```
- **To Disable**: Set both to 0ms

### TTS Context & Expressiveness (US-019 Fixes 1-4)

- **Status**: Enabled by default
- **Control**: Model-gated (v2 vs v3); see Model-Gated Features below
- **Behavior**:
  - `previous_text` / `next_text` for same-character dialogue continuity (v2 only)
  - `previous_request_ids` for acoustic chaining via sliding window (all models)
  - Aggressive segment splitting to achieve Fix 4 continuity on v2
- **To Disable**: Would require changing model selection or modifying `SegmentContextResolver`

## Model-Gated Features

These features are controlled by ElevenLabs model selection and **cannot be toggled at runtime** without changing `_MODEL_ID` in `elevenlabs_provider.py` (compile-time).

### Inline Emotion Tags

- **Feature**: `[emotion] text` syntax support
- **Gated by**: `_MODEL_CAPS["inline_tags"]`
- **Supported in**: ElevenLabs v3 only
- **Status**: NOT currently used (v2 is default model due to context_params support)
- **Impact**: v3 could use tags for richer emotion, but sacrifices context params

### ALL-CAPS Emphasis

- **Feature**: `text` treated as emphasis (prosodic prominence)
- **Gated by**: `_MODEL_CAPS["allcaps_emphasis"]`
- **Supported in**: ElevenLabs v3 only
- **Status**: NOT currently used
- **Impact**: Could emphasize key words but sacrifices context params

### Context Params (previous_text, next_text)

- **Feature**: Dialogue continuity via context
- **Gated by**: `_MODEL_CAPS["context_params"]`
- **Supported in**: ElevenLabs v2 only
- **Status**: ACTIVELY USED (key part of US-019 Fix 1-2)
- **Impact**: v2 chosen as default model because continuity is critical for audiobook quality

### Dynamic Number of Previous Request IDs

- **Feature**: Acoustic chaining via request ID sliding window
- **Gated by**: Model capability (both v2 and v3 support, configurable window size)
- **Supported in**: All models
- **Status**: ACTIVELY USED (US-019 Fix 2)
- **Impact**: Maintains voice consistency across segments

## Feature Flag Initialization

### TTSOrchestrator Constructor

```python
TTSOrchestrator(
    provider: TTSProvider,
    output_dir: Path,

    # Explicitly flagged features:
    ambient_enabled: bool = True,
    ambient_client: Optional[Any] = None,
    sound_effects_enabled: bool = True,
    sound_effect_client: Optional[Any] = None,
    emotion_enabled: bool = True,
    voice_design_enabled: bool = True,
    scene_context_enabled: bool = True,
    debug: bool = False,

    # Configurable silence durations:
    silence_same_speaker_ms: int = 150,
    silence_speaker_change_ms: int = 400,
)
```

### TTSProjectGutenbergWorkflow.run()

```python
def run(
    self,
    url: str,
    chapter_limit: int = 3,
    reparse: bool = False,
    debug: bool = False,
    # Feature flags:
    ambient_enabled: bool = True,
    sound_effects_enabled: bool = True,
    emotion_enabled: bool = True,
    voice_design_enabled: bool = True,
    scene_context_enabled: bool = True,
) -> Book:
    """
    Parameters:
    - ambient_enabled: Generate ambient background audio
    - sound_effects_enabled: Insert diegetic sound effects into silence gaps
    - emotion_enabled: Apply emotion tags to voice synthesis
    - voice_design_enabled: Call Voice Design API for custom voices
    - scene_context_enabled: Apply scene-based voice modifiers
    """
```

## Recommended Usage

### Production (High Quality, All Features)

```python
from src.tts.tts_orchestrator import TTSOrchestrator
from elevenlabs import ElevenLabsClient

client = ElevenLabsClient(api_key="...")

orchestrator = TTSOrchestrator(
    segments=...,
    character_assignments=...,
    output_dir=Path("output"),
    ambient_enabled=True,
    ambient_client=client,
    sound_effects_enabled=True,
    sound_effect_client=client,
    debug=False,
)

audio = orchestrator.synthesize_chapter(...)
```

### Development (Faster, Core Features Only)

```python
orchestrator = TTSOrchestrator(
    segments=...,
    character_assignments=...,
    output_dir=Path("output"),
    ambient_enabled=False,  # Skip ambient generation
    sound_effects_enabled=False,  # Skip sound effects generation
    debug=True,  # Keep segment files for inspection
)

audio = orchestrator.synthesize_chapter(...)
```

### Testing (Minimal Features, No API Calls)

```python
orchestrator = TTSOrchestrator(
    segments=...,
    character_assignments=...,
    output_dir=Path("output"),
    ambient_enabled=False,
    ambient_client=None,
    sound_effects_enabled=False,
    sound_effect_client=None,
    debug=False,
)

audio = orchestrator.synthesize_chapter(...)
```

## Command-Line Interface (CLI)

The feature flags can be toggled via command-line arguments when using `scripts/run_workflow.py` with the `--workflow tts` option.

### CLI Flag Format

Each feature has two flags: `--enable-<feature>` and `--disable-<feature>`. If neither is specified, the feature defaults to enabled. If both are specified, `--disable-` takes precedence.

### Available CLI Flags

```bash
# Ambient background sound
--enable-ambient          # Enable (default)
--disable-ambient         # Disable

# Sound effects
--enable-sound-effects    # Enable (default)
--disable-sound-effects   # Disable

# Emotion tags
--enable-emotion          # Enable (default)
--disable-emotion         # Disable

# Voice design
--enable-voice-design     # Enable (default)
--disable-voice-design    # Disable

# Scene context
--enable-scene-context    # Enable (default)
--disable-scene-context   # Disable
```

### CLI Examples

```bash
# All features enabled (default)
python scripts/run_workflow.py --url <url> --workflow tts

# Disable ambient and sound effects to save API costs
python scripts/run_workflow.py --url <url> --workflow tts \
  --disable-ambient --disable-sound-effects

# Core features only (voice design disabled)
python scripts/run_workflow.py --url <url> --workflow tts \
  --disable-voice-design

# Minimal setup (emotion only)
python scripts/run_workflow.py --url <url> --workflow tts \
  --disable-ambient --disable-sound-effects --disable-voice-design \
  --disable-scene-context
```

## Config File Support

Feature flags can be defined in a YAML or JSON config file. Copy `config/features.example.yaml` and modify as needed.

### YAML Config Format

```yaml
features:
  ambient_enabled: true
  sound_effects_enabled: true
  emotion_enabled: true
  voice_design_enabled: true
  scene_context_enabled: true
```

### JSON Config Format

```json
{
  "features": {
    "ambient_enabled": true,
    "sound_effects_enabled": true,
    "emotion_enabled": true,
    "voice_design_enabled": true,
    "scene_context_enabled": true
  }
}
```

### Loading Config Files from Python

```python
from src.config.feature_flags import FeatureFlags

# Load from YAML
flags = FeatureFlags.from_yaml("config/features.yaml")

# Load from JSON
flags = FeatureFlags.from_json("config/features.json")

# Pass to workflow
workflow = TTSProjectGutenbergWorkflow.create()
book = workflow.run(
    url="https://example.com/book.zip",
    chapter_limit=1,
    ambient_enabled=flags.ambient_enabled,
    sound_effects_enabled=flags.sound_effects_enabled,
    emotion_enabled=flags.emotion_enabled,
    voice_design_enabled=flags.voice_design_enabled,
    scene_context_enabled=flags.scene_context_enabled,
)
```

2. **`breath_pauses_enabled: bool = False`** — When implemented (US-017)

### Config File Support

Define feature flags in a YAML/JSON config file:

```yaml
# config/features.yaml
features:
  ambient_sound:
    enabled: true
    volume: -28
  sound_effects:
    enabled: true
  emotion_tags:
    enabled: true
  voice_design:
    enabled: true
  scene_context:
    enabled: true
  context_params:
    enabled: true
  debug_mode:
    enabled: false
```

Then load at startup:

```python
from src.config import FeatureFlags

flags = FeatureFlags.load("config/features.yaml")

orchestrator = TTSOrchestrator(
    ...,
    ambient_enabled=flags.ambient_sound.enabled,
    sound_effects_enabled=flags.sound_effects.enabled,
    debug=flags.debug_mode.enabled,
)
```
